把时间线再往前倒一下，docker是怎么开始的

发表于 2020-12-23 更新于 2020-12-24 Valine：

也写好几篇Docker了，每次开始的前提都是已经装好docker了，或者本身所使用的系统内置Docker，甚至带图形化管理工具，那如果我没有装好的系统各种NAS，我也不会安装怎么办，上次我在一篇里边写使用portainer的方法的时候，提了下加速镜像设置，还有人问具体地址是啥看不清，今天就再来篇前传。

Docker是啥东东

我从书上看的：Docker是一种运行于Linux和Windows上的软件，用于创建、管理和编排容器。

那容器又是个啥？容器其实与虚拟机啊类似，但是又不像虚拟机一样需要专门的操作操作系统。只是把应用所需的运行环境做一个最小化的打包。不会像虚拟机那样，占用多余的CPU和内存资源，找张图来做个比较

图没找到，只好自己画了一张，稍微有些难看，但不影响理解

还有Docker中常见的两个概念容器(container)和镜像(image)，简单理解，镜像是一片片的，容器是把一个或几个镜像放到里边

安装

安装建议使用Linux系统不管是虚拟机还是实体机都可以，Windows虽然也有，但后续安装镜像会有一点小问题：在设置容器内与容器外对应路径的时候。还有就是支持windows版的镜像太少了

不废话，就开始安装过程，通过一行命令完成
